Step 1 输出: 严格反应事件

相关候选过程

                  

                  
这里保留与你查询反应相关、但不满足严格条件的过程，例如 partial、multiplicity、sampling sparse 或 context fallback。它们不再冒充主结果，但仍可单独下钻核查。

                  

                  

                    

                      
                      
                    

                  

                

                

                  查看已丢弃候选 / 调试
                  

                    
这些候选已经被硬拒绝，不会进入 Step 2。常见原因是前后 comparison frame 无净变化、方向错误，或完全不支持当前查询反应。

                    

                    

                      

                        
                        
                      

                    

                  

                

              


              

                
Step 2. 选中单事件并提取局部反应轨迹

                

                  主流程只做两件事：先从 Step 1 主表里选中一条“严格反应事件”，再在这里直接生成该单事件的局部子轨迹、5 帧关键帧故事板，并一键打开 OVITO / VMD。若你是从“相关候选过程”进入，这里会明确标记为候选核查模式。手工帧范围和 atom ids 覆盖项只保留在下方高级区。
                

                

                  

                    

                      
从当前已选事件直接下钻

                      
在 Step 1 结果表中点击“选为 Step 2”后，这里会自动带入该事件的 anchor、window 和 route 解析结果，直接生成用于动态核查的事件子轨迹。

                    

                    

                      
                      
尚未选择事件。请先在上方 Step 1 的“反应事件候选实例”中载入一条事件，或在右侧填写手工切片范围。

                    
这个入口会优先使用“已选事件”的 anchor、window 和 route 解析结果，不依赖你右侧手工填写的覆盖项。

                    
默认导出的 subset 不是整盒子，也不是只有变化原子，而是围绕 reaction core 在关键三帧上取“完整分子并集”的上下文。

                  

                    

                      
                      
                      
优先显示“用于轨迹可视化的上下文原子集”，通常会比纯变化原子更完整，更适合看前后几帧的动态过程。这里给出的是原始 LAMMPS atom id，不是 VMD 的 index。

                    

                    

                      
                    

                    

                      
                      
                      
在 OVITO 中添加 Expression selection modifier，并粘贴这里的表达式，即可按原始 atom id 精确选中参与事件的原子。

使用说明

      

        

          
快速开始

          
    
            
  1. 如果要做网络检索/flux富集，先确认顶部 reactionabcd 文件 与 min_tp。
    2. 
            
  2. 在任一检索卡片输入条件并点击“查询”。
    3. 
            
  3. 结果表会自动生成结构列表；鼠标悬停在 SMILES 上可快速预览。
    4. 
            
  4. 在 时间演化绘图 选择 Species 或 Carbon 模式后绘图。
    5. 
            
  5. Carbon 模式可从 .species 直接出图并在同窗口勾选曲线对比。
    6. 
          

        

        

          
常用模块

          
    
            
  • 按分子式查 SMILES：查同分子式下的全部/Top 物种。
    • 
            
  • 按质量数查 MD 物种：按 nominal/exact 质量筛选候选物种。
    • 
            
  • 按 SMILES / 分子式查后续路径：看某物种的消耗/生成通道。
    • 
            
  • Species 中间体筛选：按先升后降 + 丰度 + FWHM 自动筛候选。
    • 
            
  • 按公式级反应检索：支持仅填反应物、仅填产物或双侧同时填写。
    • 
            
  • 事件轨迹提取：先定位物种/反应事件，再按事件窗口或手工帧范围提取 lammpstrj 子轨迹，用于在 OVITO / VMD 中检查动态过程。
    • 
            
  • 按公式级反应检索中的Top占比可直接算检索结果的占比。
    • 
          

        

        

          
输入格式示例

          
    
            
  • 分子式：C6H4、C6H4O2
    • 
            
  • SMILES：[H][O][H]
    • 
            
  • 多项输入：使用 +、, 或换行分隔。
    • 
            
  • Plot 目标支持：label::query，如 oBenzyne::C6H4。
    • 
            
  • 多文件对比：system@replicate::/abs/path.species 或 system@replicate::/abs/path.reactionabcd（每行一个）。
    • 
          

        

        

          
占比说明

          
    
            
  • Top占比基于当前反应检索结果计算，无需额外 CSV。
    • 
            
  • ShareMetric可选 net_tp / tp / reverse_tp。
    • 
            
  • ShareAbs用于按绝对值统计（适合净通量正负并存场景）。
    • 
            
  • Positive only只统计大于 0 的指标值。
    • 
            
  • 绘图模块会优先使用你填写的 .species/.reactionabcd 数据源，不强依赖顶部 reactionabcd。
